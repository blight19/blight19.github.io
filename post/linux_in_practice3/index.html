Linux是怎样工作的读书笔记-第三章

Posted by 闫的Blog on Friday, January 12, 2024

进程管理

创建进程

目的：

  • 将同一个程序分成多个进程进行处理

  • 创建另一个程序

Linux提供了两个函数：fork和execve

fork函数

父进程执行fork以后，生成一个子进程。流程如下

  1. 为自进程申请内存空间，并复制父进程的内存到子进程的内存空间。

  2. 父进程与子进程分裂成两个进程，执行不同的代码。用fork的返回值控制执行不同的代码。

#include <unistd.h>
#include <stdio.h>
#include <stdlib.h>
#include <err.h>

static void child()
{
	printf("I'm child! my pid is %d.\n", getpid());
	exit(EXIT_SUCCESS);
}

static void parent(pid_t pid_c)
{
	printf("I'm parent! my pid is %d and the pid of my child is %d.\n",
	       getpid(), pid_c);
	exit(EXIT_SUCCESS);
}

int main(void)
{
	pid_t ret;
	ret = fork();
	if (ret == -1)
		err(EXIT_FAILURE, "fork() failed");
	if (ret == 0) {
		// fork() 会返回 0 给子进程，因此这里调用 child()
		child();
	} else {
		// fork() 会返回新创建的子进程的进程 ID（大于 1）给父进程，因此这里调用 parent()
		parent(ret);
	}
	// 在正常运行时，不可能运行到这里
	err(EXIT_FAILURE, "shouldn't reach here");
}

execve函数

想要启动另外一个程序，使用execve函数。

内核在运行进程时的流程

  1. 读取可执行文件，并读取创建进程的内存映像所需信息

  2. 用新的进程数据覆盖当前进程的内存

  3. 从最初的命令开始运行新的进程

在启动另一个程序的时候，不是新增了一个进程，而是替换了当前进程。

读取可执行文件不仅包含进程在运行过程中的代码和数据还包含了开始运行时所需要的数据：

  • 代码段在文件中的偏移量，大小，以及内存映像的起始地址

  • 包含数据段在文件中的偏移量，大小，以及内存映像的起始地址

  • 程序执行的第一条指令的内存地址

基于以上信息，先将程序映射到内存上。最后从入口点开始运行程序。

Linux的可执行文件遵循的是ELF(executable and linkable format)格式，ELF的相关信息可以通过readelf命令来获得。

查看sleep这个二进制文件的起始地址

readelf -h /usr/bin/sleep

中间有一行

Entry point address:               0x2640

readelf -S /usr/bin/sleep

  • 输出的数据每两行为一组

  • 全部数据皆为16进制

  • 每组的第一行第二个字段中，.text对应代码段的信息 .data对应数据段信息

  • 第一行4字段 内存映像的起始地址 第一行第5字段 文件中的偏移量 第二行的1字段：大小

进程的内存映像信息，可以从/proc/pid/maps文件中找到

cat /proc/pid/maps

创建一个别的进程通常使用的方式是fork and exec.父进程调用fork，然后子进程调用exec

#include <unistd.h>
#include <stdio.h>
#include <stdlib.h>
#include <err.h>

static void child()
{
	char *args[] = { "/bin/echo", "hello" , NULL};
	printf("I'm child! my pid is %d.\n", getpid());
	fflush(stdout);
	execve("/bin/echo", args, NULL);
	err(EXIT_FAILURE, "exec() failed");
}

static void parent(pid_t pid_c)
{
	printf("I'm parent! my pid is %d and the pid of my child is %d.\n",
	       getpid(), pid_c);
	exit(EXIT_SUCCESS);
}

int main(void)
{
	pid_t ret;
	ret = fork();
	if (ret == -1)
		err(EXIT_FAILURE, "fork() failed");
	if (ret == 0) {
		// fork() 会返回 0 给子进程，因此这里调用 child()
		child();
	} else {
		// fork() 会返回新创建的子进程的进程 ID（大于 1）给父进程，因此这里调用 parent()
		parent(ret);
	}
	// 在正常运行时，不可能运行到这里
	err(EXIT_FAILURE, "shouldn't reach here");
}
